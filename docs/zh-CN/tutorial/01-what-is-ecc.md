# 第 1 篇：ECC 是什么

- **动手**：约 10 分钟（浏览本仓库顶层目录与两篇链接文档）

## ECC 用一句话怎么理解

**Everything Claude Code（ECC）** 是一套跑在 **AI 编程 harness**（如 Claude Code、Cursor、Codex 等）之上的工程化配置：**子代理（Agent）、技能（Skill）、斜杠命令（Command）、钩子（Hook）、规则（Rule）与 MCP 配置**被打包在一起，让你用可复用的流程做事，而不是每次都从零写一长段提示词。

版本与能力规模以根目录 [`AGENTS.md`](../../../AGENTS.md) 为准（例如当前标注的 Agent 数量、技能与命令规模）。本系列只帮你建立心智模型；具体清单会随发行版变化，请以仓库为准。

---

## 它和「只跟模型聊天」差在哪里

| 维度 | 裸聊 / 单次长提示 | ECC 一类体系 |
|------|-------------------|----------------|
| 工作流 | 靠你回忆「上次怎么做的」 | Skill / Command 把步骤固化 |
| 分工 | 一个大对话里什么都做 | 可把规划、TDD、评审交给不同 **Agent** |
| 时机 | 只有你发消息才动 | **Hook** 可在工具调用前后、停止、压缩等时机自动跑 |
| 边界 | 容易跑偏 | **Rule** + 项目 [`CLAUDE.md`](../../../CLAUDE.md) 提供长驻约束 |
| 外向能力 | 聊天框里口述「去查文档」 | **MCP** 把文档检索、浏览器等能力结构化接进来 |

这些能力并非互相替代：**Command** 是你主动点的入口；**Skill** 往往是被命令或场景拉起的说明与工作流；**Agent** 是「专才」，由主会话或编排命令委派。

更偏「体感」的说明见 [简明指南（the-shortform-guide）](../the-shortform-guide.md)。

## 六类构件与仓库里在哪里

本系列统一使用下表术语（后文不再重复定义）：

| 构件 | 中文习惯叫法 | 主要职责 | 在本仓库中的典型位置 |
|------|----------------|----------|----------------------|
| Skill | 技能 | 按需加载的流程、清单与领域知识 | [`skills/`](../../../skills/)；个人/团队扩展见 [Skill 放置策略](../../SKILL-PLACEMENT-POLICY.md) |
| Command | 斜杠命令 | 用户主动触发的快捷入口 | [`commands/`](../../../commands/) |
| Agent | 子代理 | 可被委派的专门角色（规划、TDD、评审等） | [`agents/`](../../../agents/) |
| Hook | 钩子 | 由生命周期或工具事件触发的自动化 | [`hooks/`](../../../hooks/) |
| Rule | 规则 | 与项目协同的长驻约束 | [`rules/`](../../../rules/) 与 [`CLAUDE.md`](../../../CLAUDE.md) |
| MCP | MCP 配置 | 外向工具与服务的接入形态 | [`mcp-configs/`](../../../mcp-configs/) |

此外还有 [`scripts/`](../../../scripts/)（钩子等调用的跨平台脚本）、[`tests/`](../../../tests/)（本仓库自测）。**学 ECC 不必先背全目录**，先记住上表六类即可。

## 谁能从本系列得到什么

- **初学者**：读完第 1～3 篇，能说明六类构件、完成一次本地测试、跑通一条「规划 → 实现/测试 → 评审」式的命令链思路。  
- **日常开发者**：在第 2～4、6 篇基础上，把 ECC 接到真实项目：知道命令与 Agent 去哪查表、Hook/Rule 何时介入、安全底线在哪。  
- **扩展者与贡献者**：第 7～8 篇 + 官方 [CONTRIBUTING.md](../../../CONTRIBUTING.md)，能写最小 Skill、理解向上游 PR 前要满足什么。

## 常见坑

- **把「安装 ECC / 同步本仓库」当成「装好 Claude Code 就万事俱备」**：客户端、账号与各 harness 的配置仍要以官方与 [README.zh-CN.md](../../../README.zh-CN.md) 为准；第 2 篇会拆开说。  
- **以为 Agent 越多越好**：Delegating 有价值的前提是你知道**当前任务该用哪条命令、对应哪个 Agent**——地图在 [COMMAND-AGENT-MAP.md](../../COMMAND-AGENT-MAP.md)。
