# 第 1 篇：ECC 是什麼

## 本篇要花多久

- **動手**：約 10 分鐘（瀏覽本儲存庫頂層目錄與兩篇連結文件）

## 學前需備

## ECC 用一句話怎麼理解

**Everything Claude Code（ECC）** 是一套跑在 **AI 程式設計 harness**（如 Claude Code、Cursor、Codex 等）之上的工程化設定：**子代理（Agent）、技能（Skill）、斜線命令（Command）、鉤子（Hook）、規則（Rule）與 MCP 設定** 打包在一起，讓你用可複用的流程做事，而不是每次都從零寫一大段提示詞。

版本與能力規模以根目錄 [`AGENTS.md`](../../../AGENTS.md) 為準。本系列只幫你建立心智模型；具體清單會隨發行版變化，請以儲存庫為準。

---

## 它和「只跟大家聊天」差在哪裡

| 維度 | 裸聊／單次長提示 | ECC 一類體系 |
|------|------------------|--------------|
| 工作流 | 靠你回想「上次怎麼做的」 | Skill／Command 把步驟固化 |
| 分工 | 一個大對話裡什麼都做 | 可把規劃、TDD、評審交給不同 **Agent** |
| 時機 | 只有你發訊息才動 | **Hook** 可在工具呼叫前後、停止、壓縮等時機自動跑 |
| 邊界 | 容易跑偏 | **Rule** + 專案 [`CLAUDE.md`](../../../CLAUDE.md) 提供長駐約束 |
| 外向能力 | 聊天框裡口述「去查文件」 | **MCP** 把文件檢索、瀏覽器等能力結構化接進來 |

**Command** 是你主動點的入口；**Skill** 常是被命令或場景拉起的說明與工作流；**Agent** 是「專才」，由主會話或編排命令委派。

更直觀的說明見 [簡明指南（簡體）](../../zh-CN/the-shortform-guide.md) 或 [英文簡明指南](../../../the-shortform-guide.md)。

## 六類構件與儲存庫裡在哪裡

| 構件 | 習慣叫法 | 主要職責 | 在本儲存庫中的典型位置 |
|------|----------|----------|------------------------|
| Skill | 技能 | 按需載入的流程、清單與領域知識 | [`skills/`](../../../skills/)；擴展見 [Skill 放置策略](../../SKILL-PLACEMENT-POLICY.md) |
| Command | 斜線命令 | 使用者主動觸發的捷徑入口 | [`commands/`](../../../commands/) |
| Agent | 子代理 | 可被委派的專門角色 | [`agents/`](../../../agents/) |
| Hook | 鉤子 | 由生命週期或工具事件觸發的自動化 | [`hooks/`](../../../hooks/) |
| Rule | 規則 | 與專案協同的長駐約束 | [`rules/`](../../../rules/) 與 [`CLAUDE.md`](../../../CLAUDE.md) |
| MCP | MCP 設定 | 外向工具與服務的接入形態 | [`mcp-configs/`](../../../mcp-configs/) |

另有 [`scripts/`](../../../scripts/)、[`tests/`](../../../tests/)。**學 ECC 不必先背全目錄**，先記住上表六類即可。

## 誰能從本系列得到什麼

- **初學者**：讀完第 1～3 篇，能說明六類構件、完成一次本機測試、跑通「規劃 → 實作／測試 → 評審」式命令鏈思路。  
- **日常開發者**：在第 2～4、6 篇基礎上，把 ECC 接到真實專案。  
- **擴展者與貢獻者**：第 7～8 篇 + [CONTRIBUTING](../../../CONTRIBUTING.md)。

## 常見坑

- **把「安裝 ECC／同步本儲存庫」當成「裝好 Claude Code 就萬事俱備」**：仍須以官方與 [README.zh-CN.md](../../../README.zh-CN.md) 或本倉英文 README 為準；第 2 篇會拆開說。
